1. The Problem

By default, width and height only apply to content, making sizing unpredictable.

width: 200px
Content
+ 40px padding
+ 20px border
= 260px total!
‚ö†Ô∏è Surprise! Setting width: 200px doesn't mean the element is 200px wide!

2. content-box (Default)

Width and height apply ONLY to the content area.

width: 200px
Content: 200px
+ Padding: 40px
+ Border: 20px
Total: 260px
.box {
  box-sizing: content-box; /* default */
  width: 200px;
  padding: 20px;
  border: 10px solid blue;
}

/* Total width = 200 + 40 + 20 = 260px */

3. border-box (Recommended)

Width and height include padding and border.

width: 200px
Total: 200px
Content: 140px
Padding: 40px (included)
Border: 20px (included)
.box {
  box-sizing: border-box;
  width: 200px;
  padding: 20px;
  border: 10px solid blue;
}

/* Total width = 200px (as specified!) */
Much better! The element is exactly 200px wide, as expected.

5. Best Practice: Universal Border-Box

Apply border-box to all elements for predictable sizing.

/* Add this to the top of your CSS */
*,
*::before,
*::after {
  box-sizing: border-box;
}
‚úÖ Recommended: This is considered a CSS best practice and is used in most modern projects.
