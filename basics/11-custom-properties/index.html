1. Defining Custom Properties

Define variables in :root to make them available globally.

:root {
  --primary-color: #667eea;
  --secondary-color: #764ba2;
  --spacing: 1.5rem;
  --border-radius: 8px;
}
Syntax: Custom properties start with -- and are accessed with var()

2. Using Custom Properties

Reference variables using the var() function.

Themed Box

This box uses custom properties for colors, spacing, and border-radius.

.themed-box {
  background: var(--primary-color);
  color: white;
  padding: var(--spacing);
  border-radius: var(--border-radius);
}

3. Scoped Custom Properties

Override custom properties within specific elements to create variations.

This uses the global primary color (blue)

This container redefines --primary-color to green

This container redefines --primary-color to orange

.scoped-container {
  --primary-color: #667eea; /* default */
}

.theme-green {
  --primary-color: #27ae60; /* override */
}

.theme-orange {
  --primary-color: #e67e22; /* override */
}

.scoped-text {
  color: var(--primary-color);
}
Key Point: Custom properties cascade and can be overridden in child elements.

4. Fallback Values

Provide fallback values in case a custom property is not defined.

This uses a fallback value

.fallback-demo {
  /* If --undefined-color doesn't exist, use #999 */
  color: var(--undefined-color, #999);

  /* Can also reference another variable as fallback */
  background: var(--custom-bg, var(--primary-color));
}

5. Real-World Example: Theme System

Light Theme

Background: var(--bg-color)

Text: var(--text-color)

Dark Theme

Background: var(--bg-color)

Text: var(--text-color)

.light-theme {
  --bg-color: #ffffff;
  --text-color: #2c3e50;
  --accent-color: #667eea;
}

.dark-theme {
  --bg-color: #2c3e50;
  --text-color: #ecf0f1;
  --accent-color: #3498db;
}

.theme-card {
  background: var(--bg-color);
  color: var(--text-color);
}

.theme-button {
  background: var(--accent-color);
}
