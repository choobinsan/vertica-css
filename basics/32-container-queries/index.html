1. The Problem with Media Queries

Media queries respond to the viewport size, not the container size. This makes it hard to create truly reusable components.

Example: A card component might be in a narrow sidebar or a wide main area. Media queries can't tell the difference - they only know the viewport size.

2. Container Queries to the Rescue!

Container queries let components respond to their container's size, making them truly modular and reusable.

.container {
  container-type: inline-size;
  /* or container-name: sidebar; */
}

@container (min-width: 400px) {
  .card {
    display: grid;
    grid-template-columns: 150px 1fr;
  }
}

4. Setting Up Container Queries

1
Define the container 
.container {
  container-type: inline-size;
}
2
Write container query 
@container (min-width: 400px) {
  .card {
    /* styles */
  }
}
3
Optional: Name it 
.sidebar {
  container-name: sidebar;
}

@container sidebar (min-width: 300px) {
  /* styles */
}

5. Container Types

inline-size

Queries the inline dimension (width in horizontal writing modes)

Most Common
size

Queries both inline and block dimensions

Both Axes
normal
