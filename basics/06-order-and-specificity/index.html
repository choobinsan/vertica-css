CSS Order & Specificity

Understanding how CSS determines which styles to apply ⚖️

1. The Specificity Hierarchy

When multiple rules target the same element, specificity determines the winner.

This paragraph has multiple competing styles!

Highest ID Selector #main-text { color: green; } ✓ WINS
Compound Selector p.highlight { color: orange; }
Class Selector (later) .highlight { color: purple; }
Class Selector (earlier) .highlight { color: blue; }
Lowest Type Selector p { color: red; }
Result: The text is green because the ID selector (#main-text) has the highest specificity.

2. Specificity Calculation

Specificity is calculated as a three-part value: (IDs, Classes, Elements)

p
(0, 0, 1)

1 element

.highlight
(0, 1, 0)

1 class

p.highlight
(0, 1, 1)

1 class + 1 element

#main-text
(1, 0, 0)

1 ID

div p.highlight
(0, 1, 2)

1 class + 2 elements

#main-text.highlight
(1, 1, 0)

1 ID + 1 class

3. Source Order Matters

When specificity is equal, the last rule wins.

This text demonstrates source order

/* First rule */
.order-demo {
  color: blue;
}
✗ Loses 
/* Second rule (same specificity) */
.order-demo {
  color: purple;
}
✓ Wins
Key Point: Both rules have the same specificity (0, 1, 0), so the later rule wins.

4. Best Practices

✅
Use classes for styling

Classes are reusable and have good specificity

⚠️
Avoid IDs for styling

IDs have very high specificity and are hard to override

✅
Keep specificity low

Easier to maintain and override when needed

❌
Avoid !important

Only use as a last resort; makes debugging difficult
