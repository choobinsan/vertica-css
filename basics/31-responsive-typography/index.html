1. The Problem

Fixed font sizes don't work well across different screen sizes. Text that's readable on desktop might be too small on mobile, or too large on huge monitors.

‚ö†Ô∏è Avoid this: Using fixed pixel sizes like font-size: 16px; everywhere makes text inflexible.

2. Media Query Approach

Use media queries to change font sizes at different breakpoints.

Media Query Heading

This text changes size at specific breakpoints. Try resizing your browser! 

h1 {
  font-size: 24px; /* Mobile */
}

@media (min-width: 768px) {
  h1 {
    font-size: 32px; /* Tablet */
  }
}

@media (min-width: 1024px) {
  h1 {
    font-size: 48px; /* Desktop */
  }
}
Pros: Precise control at each breakpoint
Cons: Jumps between sizes, requires multiple breakpoints

3. Viewport Units (vw)

Use viewport width units for fluid scaling.

Viewport Width Heading

This text scales smoothly with the viewport width.

h1 {
  font-size: 5vw; /* 5% of viewport width */
}
‚ö†Ô∏è Problem: Can become too small on mobile or too large on desktop without limits.

4. clamp() - The Modern Solution ‚ú®

The clamp() function provides fluid typography with minimum and maximum limits.

clamp() Heading

This text scales fluidly but never gets too small or too large. Perfect! 

h1 {
  font-size: clamp(24px, 5vw, 64px);
  /*             min   ideal  max */
}
How it works:
‚Ä¢ 24px = minimum size (mobile)
‚Ä¢ 5vw = ideal size (scales with viewport)
‚Ä¢ 64px = maximum size (large screens)

6. Practical Examples

Body Text

body {
  font-size: clamp(
    16px,
    1rem + 0.5vw,
    20px
  );
}

Headings

h1 {
  font-size: clamp(
    2rem,
    5vw,
    4rem
  );
}

Line Height

p {
  line-height: clamp(
    1.5em,
    1.5em + 0.5vw,
    2em
  );
}

Letter Spacing

h1 {
  letter-spacing: clamp(
    -0.02em,
    -0.01em + 0.1vw,
    0.02em
  );
}
